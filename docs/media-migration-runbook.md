# Runbook — סיום מיגרציית מדיה (WordPress → Supabase)

מטרה: להעביר את הקבצים שעדיין "תקועים" על השרת הישן אל הבאקט `media`, ואז לשכתב
את כתובות המסד — כדי להתנתק לגמרי מ-WordPress ומהשרת הישן.

## מצב נוכחי (נכון ל-2026-06-13)
- בבאקט `media` כבר ~88,818 קבצים (תעתיק לטיני, מהרצה קודמת).
- במסד עדיין **4,028** קבצי מדיה ייחודיים מוזכרים עם כתובת `wp-content` (ב-`image_url` וב-`content`).
- מתוכם **~1,027 כבר קיימים** בבאקט (בשם עברי) ו-**~3,001 חסרים** — אלה ה"תקועים".
- רשת ביטחון: הפרוקסי `wp.sod1820.co.il` עדיין מגיש את הקבצים העבריים ב-200, אז כל עוד
  כתובת לא שוכתבה — היא ממשיכה לעבוד.

## למה הגישה הזו (שם עברי 1:1, לא תעתיק)
השמות בבאקט הקיים הם תעתיק (`מבצע-עלות-השחר` → `mbtza-alvt-hshchr`), אבל פונקציית
התעתיק אינה בידינו וקשה לשחזרה במדויק (ח לפעמים `ch` ולפעמים `h` וכו'). לכן מושכים
כל קובץ מהפרוקסי ושומרים בבאקט **באותו שם עברי בדיוק** — כך שכתוב המסד הוא מיפוי
ודאי 1:1, בלי ניחושים ובלי סיכון לתמונות שבורות.

## שלב 1 — העלאת הקבצים החסרים (רץ אצלך / בסוכן הדפדפן)
דורש מפתח `service_role` (Supabase → Project Settings → API → `service_role`).
ה-anon **לא** יכול לכתוב לאחסון.

```bash
# מתוך תיקיית הריפו, על הענף claude/google-migration-runbook-7vxf8p
npm install
export SUPABASE_SERVICE_KEY="<service_role key מה-Supabase Dashboard>"

# תצוגה מקדימה בלבד (לא מעלה כלום) — מראה כמה חסר:
node migrate-media.mjs --dry

# העלאה בפועל (idempotent — אפשר להריץ שוב בבטחה):
node migrate-media.mjs
```

הסקריפט:
- קורא מהמסד בדיוק אילו קבצים נחוצים (לא סורק את כל ה-FTP).
- מדלג על מה שכבר בבאקט (בדיקת HEAD).
- מושך מ-`https://wp.sod1820.co.il` ומעלה ל-`media` באותו נתיב.
- שומר `migration-log.json` — כולל רשימת ה-**404** (קבצים שבאמת חסרים בשרת הישן).

כשמסיים: לחזור ולומר **"הקבצים עלו"**, ולצרף את שורת הסיכום + רשימת ה-404 מה-log.

## שלב 2 — שכתוב כתובות המסד (רק אחרי אימות, נעשה בזהירות עם גיבוי)
**לא להריץ לבד.** אחרי שהקבצים עלו, מבצעים:
1. גיבוי: `create table posts_img_backup as select wp_id, image_url, content from posts;`
2. החלפת הקידומת `https://sod1820.co.il/wp-content/uploads/` →
   `https://linswmnnkjxvweumprav.supabase.co/storage/v1/object/public/media/uploads/`
   ב-`image_url` וב-`content` (כתובות מקודדות-URL נפתרות אוטומטית מול שמות העברית בבאקט).
3. לא לשכתב קבצים שברשימת ה-404 (הם חסרים גם בשרת — נטפל בהם בנפרד).

## בטיחות
לא לפרק את WordPress / לבטל את הפרוקסי עד שכל הכתובות שוכתבו ואומתו.
