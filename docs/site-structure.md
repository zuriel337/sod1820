## 3. בעיות מבניות

1. **דפי קורסים / אודות / תשלום ללא URL** — נשמרים רק ב‑`state` בריאקט.
   תוצאה: אי אפשר לשתף קישור, אי אפשר להוסיף למועדפים, וגוגל לא מאנדקס אותם.
   לאתר שמוכר קורסים — בעיה קריטית ל‑SEO ולשיווק.
2. **הראוט `/:slug` בולעני** — כל כתובת חד-מקטעית נחשבת לסלאג של פוסט,
   מתנגש עם דפים אחרים ושביר.
3. **כתובות בעברית** (`/צור-קשר`, `/דף-צאט-ראשי`) — בעיות קידוד ושיתוף.
4. **דפי admin / theme-preview** מעורבבים באותו catch-all של תוכן ציבורי.

## 4. המבנה המוצע

היררכיה נקייה, כל דף עם URL אמיתי וידידותי ל‑SEO:

```
/                          דף הבית
│
├── /courses               כל הקורסים
│   ├── /courses/:slug          עמוד קורס בודד
│   └── /courses/:slug/checkout תשלום
│
├── /blog                  פוסטים  (במקום /post)
│   ├── /blog/:slug             פוסט בודד
│   ├── /blog/category/:slug    קטגוריה
│   └── /blog/tag/:slug         תגית
│
├── /gematria              מחשבון / חיפוש גימטריה
│   └── /gematria/:phrase       תוצאה לפי ביטוי
│
├── /about                 אודות צוריאל פולייס
├── /contact               צור קשר  (במקום /צור-קשר)
├── /chat                  צאט האתר
├── /login                 כניסה / אזור אישי
└── /admin                 ניהול (מוגן)
```

### יתרונות
- כל דף נגיש לגוגל (SEO) — חשוב לאתר מכירת קורסים.
- קישורים ניתנים לשיתוף ולהוספה למועדפים.
- היררכיה ברורה: `/courses/...`, `/blog/...`.
- ביטול כתובות עבריות בעייתיות.
- בסיס נוח להרחבה עתידית (למשל אזור תלמיד `/my-courses`).

### מסלול מעבר מומלץ (ללא שבירת קישורים קיימים)
- להפנות (redirect 301) `/post` → `/blog`, `/צור-קשר` → `/contact`.
- להמיר את דפי ה‑`state` (courses/about/checkout) ל‑`<Route>` אמיתיים.
- להוציא `/:slug` תחת `/blog/:slug` כדי לבטל את ההתנגשות.
