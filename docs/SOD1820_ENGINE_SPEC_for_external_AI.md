# SOD1820 — מפרט מנוע גימטריה + חוקי ניתוח (להזנה למנוע AI חיצוני / Gemini)

> מסמך זה הוא **מקור-האמת** להזנה למנוע ניתוח חיצוני (Gemini) שמתחבר למחשבון הקהילתי של SOD1820.
> הוא מבוסס ישירות על המנוע הרשמי (`src/lib/gematria.js`) ועל החוקים הנעולים ב-DB (`nodes type='rule'`).
> **עיקרון-על:** המנוע מחשב עובדות. ה-AI **מפרש בלבד** — לעולם לא מחשב גימטריה בעצמו.

## חלק 0 — חוקי-ברזל למנוע החיצוני (לפני הכל)

1. **אסור לחשב גימטריה מהזיכרון/ניחוש.** כל ערך מגיע מהמנוע הרשמי. אם קיבלת ערך ב-`facts` — הוא **נתון מערכת** מאומת, אל תחשב מחדש (`verified_value_is_system_data`, `gematria_engine_law`).
2. **הפרד עובדה מפרשנות.** ערך גימטריה = עובדה מדידה. משמעות/רמז = פרשנות משלימה, תמיד מסומנת ככזו.
3. **בלי נבואות, בלי תאריכים עתידיים, בלי טענות על אנשים חיים.** עברית בלבד. בלי Markdown בפלט למשתמש.
4. **רגיל הוא היסוד.** כל שיטה אחרת = תוספת. אין לבנות מסקנה מרכזית על אתב"ש/צופן לבד (`method_priority`, `method_hierarchy_ragil_foundation`).
5. **שוויון = הלב.** תובנה חזקה = שני ביטויים שונים באותו ערך (למשל 256 = «מה פעל אל» = «אהרן»). ערך בודד ללא מקבילה = חלש (`gematria_pair_law`).
6. **אנגרמה אינה חידוש** בשיטות חיבוריות (רגיל/מילוי/קדמי): אותן אותיות בסדר אחר = אותו ערך, זה טריוויאלי. חריג: שיטות צופן (אתב"ש/אלב"ם/אותיות-אחרי) — שם הסדר משמעותי.

## חלק 2 — 13 השיטות הראשיות (METHODS) + 6 מנועי עומק

לכל שיטה: **שם · תת-כותרת (חובה להצגה) · נשמה (פרשנות) · אלגוריתם מדויק**.

### שיטות ראשיות
1. **רגיל** — "חיבור ערכי האותיות" — *המהות הגלויה*. `Σ GEM[c]`.
2. **מילוי** — "ערך שֵם האות המלא" — *הפנימיות*. `Σ MILUI[c]` (א=111, ב=412, ג=83, ד=434, ה=15, ו=22, ז=67, ח=418, ט=419, י=20, כ=100, ל=74, מ=80, נ=106, ס=120, ע=130, פ=81, צ=104, ק=186, ר=510, ש=360, ת=416).
3. **מסתתר** — "הפרשים בין אותיות" — *מה שמסתתר בין האותיות*. **מילה-מילה** (הרווח שובר את הרצף!): לכל מילה `Σ |GEM[cᵢ] − GEM[cᵢ₊₁]|`, ואז סוכמים מילים. דוגמה: «משיח בן דוד» = 552+48+4 = 604.
4. **קדמי (=משולש=פוטנציאל)** — "סכום מצטבר עד האות" — *השורש המצטבר*. `Σ KID[c]` (א=1, ב=3, ג=6, ד=10, ה=15, ו=21, ז=28, ח=36, ט=45, י=55, כ=75, ל=105, מ=145, נ=195, ס=255, ע=325, פ=405, צ=495, ק=595, ר=795, ש=1095, ת=1495).
5. **ריבוע** — "סכום הקידומות המצטברות" — *ההתפשטות*. **מילה-מילה**: לכל מילה סכום הרגיל של כל הקידומות ההדרגתיות. דוד = ד(4)+דו(10)+דוד(14) = 28. גג=9, צוריאל=1432.
6. **גדול** — "סופיות 500–900" — *ההתפשטות הגדולה*. `Σ (FINAL[c] || GEM[c])`.
7. **סידורי** — "מיקום האות 1–22" — *הסדר*. `Σ ORD[c]` (א=1…ת=22, סופיות=בסיס).
8. **אתבש** — "היפוך הא״ב" — *המראה*. `Σ ATB[c]` (א↔ת: א=400, ב=300…ת=1).
9. **אלבם** — "חצי מול חצי" — *הזיווג המשלים*. `Σ ALB[c]`.
10. **אותיות אחרי** — "כל אות → הבאה בא״ב (צופן המזוזה)" — *הצעד קדימה*. כל אות→הבאה (א→ב…ת→א מעגלי), ואז **רגיל של המילה החדשה**. יהוה→כוזו. סופיות מנורמלות לבסיס.
11. **אותיות לפני** — "כל אות → הקודמת בא״ב" — *הצעד אחורה*. כל אות→הקודמת (מעגלי), ואז רגיל.
12. **מילוי בלבד** — "המילוי פחות האות עצמה" — *הנסתר שבאות*. `Σ MILUI[c] − Σ GEM[c]`.
13. **הכפלה** — "כל אות בריבוע (אות×עצמה)" — *העוצמה הפנימית*. `Σ GEM[c]²`. בינה=2629. סופיות=רגיל.

### מנועי עומק (שכבה 2)
14. **משולש גדול** (קדמי גדול) — סדר מורחב עם סופיות ך=1995, ם=2595, ן=3295, ף=4095, ץ=4995. «שובו בנים שובבים»=7760. *רק אם יש סופית* (אחרת = קדמי רגיל).
15. **מסתתר גדול** — מסתתר על ערכי גדול (מילה-מילה). מלך=480, תורה=783, ירושלים=1558.
16. **מילוי דמילוי** — מילוי של המילוי. יהוה=610.
17. **מילוי דמילוי גדול** — סופיות פנימיות בערכי גדול. ירושלים=6770, יהוה=610.
18. **הכפלה גדולה** — ריבוע עם סופיות גדולות (ך=500²). מלך=252500.
19. **ריבוע גדול** — ריבוע הקידומות בסופיות גדול (מילה-מילה).

## חלק 3 — כלל «גדול = רגיל כשאין סופיות» (`gadol_equals_ragil_when_no_sofiot`)

ההבדל היחיד בין כל שיטה-«גדולה» לבסיסית = ערכי חמש הסופיות. **אם בביטוי אין ך/ם/ן/ף/ץ בצורתן הסופית → הגדול = הבסיס בדיוק.** אסור להציג את הגדול כ«ממצא נפרד» — לציין «≡ זהה לרגיל». חריג: «מילוי דמילוי גדול» (נבדק על המילוי הפנימי).
זוגות: גדול=רגיל · מסתתר גדול=מסתתר · ריבוע גדול=ריבוע · משולש גדול=קדמי · הכפלה גדולה=הכפלה.

## חלק 5 — פורמט הפלט שהמנוע החיצוני מקבל (חוזה `ai-analyze`)

הקריאה למנוע (מתאימה גם ל-Gemini):
```
analyze({ kind, subject, facts, again, fast })
```
- `kind`: `number` (עיקרי במחשבון) · `compare` · `notarikon` · `verse` · `research`.
- `subject`: הביטוי/המספר שנבחר.
- `facts`: **מערך העובדות המאומתות מהמנוע** — כל השיטות + ערכיהן + מקבילות באותו ערך. ה-AI מפרש רק את אלה.
- `again=true`: זווית נוספת. `fast=true`: מודל מהיר לכלים אינטראקטיביים (במחשבון תמיד `fast:true`).

**דוגמת `facts` שתוזן ל-Gemini:**
```json
{
  "subject": "מה פעל אל",
  "methods": [
    {"key":"רגיל","value":256,"parallels":["אהרן"]},
    {"key":"מסתתר","value":...},
    {"key":"קדמי","value":...}
  ],
  "note": "כל ערך אומת במנוע הרשמי — אל תחשב מחדש."
}
```

**כללי הפלט של ה-AI (SYSTEM):** מפרש רק עובדות שסופקו · לא מחשב · מפריד עובדה מפרשנות · בלי נבואות/תאריכים-עתידיים/טענות על חיים · עברית · בלי Markdown.

### פלט הפרשנות המצופה מ-Gemini (דוגמה, עברית, בלי חישוב, עובדה≠רמז)
> **עובדה:** «אברהם» = 248 (רגיל), וכך גם «רחם» ו«במדבר».
> **רמז משלים:** קו שמחבר את אברהם — אבי-האומה — עם «רחם» (חסד, ראשית החיים) ועם «במדבר» (מקום ההתהוות של העם). ההצעה הפרשנית ניתנת ככיוון-מחשבה, לא כקביעה.
> **הערה מבנית:** הערך «גדול» (808) שונה מהרגיל בגלל האות הסופית ם — זו אינה שיטה נוספת אלא אותה מילה עם ערך-סופית.

## חלק 7 — איך מחברים את Gemini בפועל (הצעת ארכיטקטורה)

המנוע הקיים = Edge Function `ai-analyze` (Claude). כדי להוסיף את Gemini כ**מנוע נוסף** באותו דפוס:
1. Secret חדש ב-Edge: `GEMINI_API_KEY` (לא Vault — כמו `ANTHROPIC_API_KEY`).
2. פרמטר `engine: 'claude' | 'gemini'` ב-`getAiAnalysis`; ברירת-מחדל נשארת Claude.
3. אותו `SYSTEM prompt` (חוקי-הברזל מחלק 0+5) לשני המנועים — כדי שהניתוח יהיה עקבי.
4. אותו CORS קנוני (חובה `x-client-info` + `x-supabase-api-version`, אחרת הדפדפן חוסם בשקט).
5. במחשבון הקהילתי: כפתור «נתח ב-Gemini» לצד «נתח ב-Claude», או השוואה זו-לצד-זו.

> החוזה המלא במערכת: `select description from nodes where rule_id='ai_analyze_contract';`
