## 3. תוכנית עבודה 30 יום — לפי ROI

### שבוע 1 (ימים 1–5) — "ניצחונות מהירים" · נתונים חיים · בלי פריסת Vercel
- B1: `integer→bigint` על עמודות השיטות + `bidim.value` + `all_values` + פונקציות SQL.
- B2: `GRANT SELECT` ל-anon על `gematria_wall`/`word_aliases`; guard בלקוח נגד כתיבת `user_activity` לאנונימי.
- B5: ביטול פרסום Realtime ל-`subscribers`/`edges`/`entities`.
- B6: הסרת אינדקסים כפולים/לא-בשימוש + הוספת 12 אינדקסי FK.
- B9 (חלק זול): עטיפת `(select auth.uid())` ב-41 מדיניויות.
- **ROI:** מתקן פיצ'ר ליבה, משתיק את שגיאת ה-DB מס' 1, מוריד ~מייד 10–15% מזמן ה-DB. סיכון אפסי.

### שבוע 2 (ימים 6–12) — הלב של הסקייל · פריסת קוד ראשונה
- B4: RPC יחיד לספירות bidim במחשבון (במקום 8 קריאות) + שכבת cache/dedup לקריאות החמות.
- B8: החלפת `getDistinctCategoriesAndTags` ב-RPC (מתקן אובדן קטגוריות → שלמות עמודי-תיוג ל-SEO).
- B7: הוצאת קריאות API חיצוני מה-DB ל-Edge; הסרת `pg_timezone_names`.
- **ROI:** חיתוך הצוואר הגדול ביותר (bidim 16.5%) → תשתית ל-100K משתמשים; הפחתת egress/reads.

### שבוע 3 (ימים 13–20) — מובייל / SEO / המרה
- B3: פיצול `supabase.js`, הפסקת ייבוא-סטטי של `legacy.jsx`, תיקון `AuthPage`, חילוץ `POST_CONTENT_CSS` ל-CSS אמיתי.
- מדידת Core Web Vitals לפני/אחרי (LCP מובייל).
- **ROI:** דירוג גוגל + הפחתת נטישה במובייל = יותר טראפיק אורגני והמרות שער-הרשמה.

### שבוע 4 (ימים 21–30) — הקשחה, יסודות, והשוואה לבסיס
- B10: `search_path=''`, `REVOKE EXECUTE … FROM anon`, `security_invoker` ל-2 views, סגירת listing ל-buckets, הפעלת HIBP.
- B13: אימוץ `supabase/migrations`.
- B14: ארכוב/מחיקת 17 טבלאות backup.
- B12: תכנון העברת egress תמונות ל-R2.
- **סיום:** הרצת בדיקת בריאות חדשה והשוואה מול Baseline זה.
