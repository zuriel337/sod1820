# הגדרת מודרציה וצביעה לצ'אט Spot.IM / OpenWeb

מסמך זה מרכז את כל מה שצריך להגדיר **בלוח הבקרה של Spot.IM** (https://admin.spot.im) — דברים שאי אפשר לעשות בקוד האתר, כי הצ'אט רץ בתוך iframe / shadow-DOM ש-CSS חיצוני לא חודר אליו.

- **Spot ID:** `sp_OVtajBTj`
- דף הצ'אט באתר: `/community/chat` (רכיב `SpotimChatPage` ב-`src/legacy/legacy.jsx`).
- הרכיב הנוסף `SpotimComments` מטמיע צ'אט מתחת לפוסטים בודדים (אותו Spot ID).

> שמות התפריטים ב-admin.spot.im עשויים להשתנות מעט בין גרסאות הדאשבורד. אם תווית מסוימת לא נמצאת בדיוק בשם המופיע כאן — חפש את האפשרות הקרובה ביותר באותו מסך (Settings / Moderation / Appearance).

---

## 1. רשומים בלבד יכולים לכתוב (חסימת אנונימיים)

מטרה: גולש שלא נרשם/לא מחובר לא יוכל לפרסם תגובה — רק לקרוא.

מסלול בתפריט:

1. היכנס ל-https://admin.spot.im והתחבר לחשבון שמנהל את `sod1820`.
2. בחר את ה-Spot הרלוונטי (`sp_OVtajBTj`).
3. `Settings` → `Conversation` (או `General` / `Login & Registration`, תלוי בגרסה).
4. חפש את המתג **"Require login to comment"** / **"Require registration before commenting"** / **"Disable guest commenting"** והפעל אותו.
5. אם מופיע מצב **Guest commenting** (תגובות אורח) — כבה אותו (Off).
6. שמור (`Save`).

אם החשבון משתמש ב-SSO (התחברות דרך האתר עצמו) — תחת `Settings` → `Login / SSO` מגדירים שספק ההתחברות הוא חובה. באתר שלנו ההתחברות מנוהלת ב-Supabase Auth; אם רוצים לאחד את ההתחברות, צריך להגדיר SSO מול Spot.IM בנפרד (אינטגרציה צד-שרת — לא חלק מהמסמך הזה).

תוצאה: כפתור "הוסף תגובה" יציג למשתמש לא-מחובר בקשת התחברות/הרשמה לפני שליחה.

---

## 2. אישור ידני לכל הודעה (Pre-Moderation)

מטרה: כל הודעה (ובמיוחד ממשתמשים חדשים) ממתינה לאישור מנהל לפני שתופיע בפומבי.

מסלול בתפריט:

1. ב-admin.spot.im → `Moderation` (לעיתים `Settings` → `Moderation`).
2. חפש את הגדרת **Pre-Moderation** / **"Hold comments for review"** / **"Review before publishing"**.
3. בחר את רמת ההחזקה:
   - **כל ההודעות** — אם רוצים שכל תגובה תמתין לאישור (הכי מחמיר; מתאים לדרישה של צוריאל).
   - **משתמשים חדשים בלבד** — אם Spot.IM מציע Pre-Moderation לפי trust-level, זו האפשרות הקרובה ביותר ל"משתמשים חדשים עוברים ביקורת". בחר בה אם אין אפשרות "כל ההודעות" או אם רוצים פחות עומס ניהול.
4. הפעל גם **Profanity filter** / סינון אוטומטי אם זמין, כשכבה נוספת.
5. שמור.

ניהול התור: התגובות הממתינות מופיעות תחת `Moderation` → `Pending` / `Needs review`. שם מאשרים (`Approve`) או דוחים (`Reject`) כל הודעה.

> הערה: ב-Spot.IM/OpenWeb הזמינות המדויקת של "Pre-Moderation לכולם" מול "לחדשים בלבד" תלויה בתוכנית/בגרסת הדאשבורד. אם "כל ההודעות" לא זמין — השתמש ב"משתמשים חדשים בלבד" + סינון מילים אוטומטי כפתרון הקרוב ביותר, כפי שהמשתמש ביקש.

## 3. צבעי כפתורים / מותג (Theme / Appearance)

מטרה: כפתורי הצ'אט ("הוסף תגובה", "שלח", קישורים) יהיו בצבעי הזהב/סגול של האתר.

מסלול בתפריט:

1. ב-admin.spot.im → `Settings` → `Appearance` / `Theme` / `Design`.
2. הזן את צבעי המותג. ערכי הצבע נלקחים ישירות מאובייקט `C` ב-`src/legacy/legacy.jsx`:

| תפקיד | טוקן בקוד (`C`) | ערך HEX |
|---|---|---|
| כפתור ראשי / Accent (זהב) | `C.gold` | `#d4af37` |
| הדגשה בהירה / Hover (זהב בהיר) | `C.goldBright` | `#f6e27a` |
| זהב עמום / מסגרות | `C.goldDim` | `#9a7818` |
| זהב בהיר משני | `C.goldLight` | `#e8c840` |
| רקע כהה (טקסט על כפתור זהב) | `C.bg` | `#07050E` |
| רקע משטח | `C.surface` | `#0d0a0e` |
| הדגשה מלכותית (סגול) | `C.royal` | `#3d1f5c` |
| סגול בהיר | `C.royalLight` | `#6b3fa0` |
| אדום-ארגמן (משני/התראה) | `C.crimson` | `#7a1320` |

3. המלצה למיפוי:
   - **Primary / Accent / Button color** → `#d4af37` (זהב).
   - **Button text color** → `#07050E` (רקע כהה, ניגודיות גבוהה על הזהב).
   - **Hover / Highlight** → `#f6e27a`.
   - **Background / Surface** (אם ניתן להגדיר Dark theme) → `#07050E` / `#0d0a0e`.
   - **Link color** → `#d4af37`.
   - אם יש שדה לצבע מותג משני — `#3d1f5c` (הסגול המלכותי).
4. אם קיים מתג **Dark mode** — הפעל אותו כדי שהרקע יתאים לאתר הכהה.
5. שמור ובדוק בתצוגה מקדימה.

## מה כבר נעשה בקוד (לא דורש דאשבורד)

- **טאבי הניווט של דף הצ'אט** ("צאט האתר" וכו') כבר משתמשים בצבעי האתר (`C.goldBright`, `C.gold`, `C.muted`) — אין צורך בשינוי.
- **CSS override חיצוני** נוסף ב-`SpotimChatPage` (`src/legacy/legacy.jsx`) שמכוון לסלקטורים הציבוריים של Spot.IM (`[data-spotim-module] .spcv_*`, `[class*="send-button"]` וכו') ודוחף את צבעי הזהב. **מגבלה:** ה-CSS הזה עובד רק על אלמנטים שאינם בתוך shadow-DOM/iframe; לכן הצביעה המלאה של כפתורי הצ'אט הפנימיים חייבת להיעשות בדאשבורד (סעיף 3 לעיל).
