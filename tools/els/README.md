# הצופן התנכי — כלי דילוגי-האותיות (ELS)

מקור-האמת של הכלי **«הצופן התנכי»** שרץ ב-`sod1820.co.il/code` וגם בהיכל (`/research?tool=els`).

## מה זה
כלי **עצמאי (vanilla-JS, קובץ יחיד)** לחיפוש דילוגי-אותיות בכל התנ״ך. הפלט הבנוי הוא
`public/tzofen.html` (~2.2MB) — קובץ מלא ללא תלות-רשת. כל התנ״ך (1,204,583 אותיות, 24 ספרים)
דחוס gzip+base64 ומפוענח בדפדפן ע״י `DecompressionStream` (בלי ספרייה, תואם-CSP).

## איך זה מוטמע באתר
`public/tzofen.html` מוצג כ-**iframe** דרך הרכיב `src/components/TzofenEmbed.jsx`
(אותו דפוס כמו `public/heichal.html`). מוצג ב:
- `src/pages/CodePage.jsx`  → `/code` (עמוד מלא)
- `src/pages/ResearchPage.jsx` → `/research?tool=els` (בהיכל/סביבת-המחקר)

הגשר iframe↔אתר (ב-`TzofenEmbed`): (1) שולח דרגת-משתמש (`postMessage tier`), (2) רושם כל חיפוש
דרך `track("els",…)` הקיים → `events`+`visitor_events`, (3) מציג `SubscribeGate` בשער-ההרשמה.

## «מי נגד מי» — שני מימושי-ELS בקוד (לא למחוק בטעות!)
1. **הכלי הזה (`public/tzofen.html`) = הקנוני.** מה שהמשתמשים רואים ב-`/code` + היכל.
2. **`src/components/ElsGrid.jsx` (React) — מנוע ה-ELS הישן/של סוכן קודם. נשמר, לא נמחק.**
   מוצג ב-`/code/ארכיון` (`src/pages/CodeArchivePage.jsx`). היסטוריית-git שלו (ענף
   `claude/els-research-lab` / PR #101) **לא-קשורה** ל-main (unrelated histories) — אי-אפשר
   למזג. הכלי החדש החליף אותו ברינדור הקנוני; הישן נשמר כארכיון בלבד.

## בנייה / עריכה
```bash
cd tools/els
python3 build.py            # → כותב ../../public/tzofen.html (דטרמיניסטי)
```
- **לערוך את הכלי:** משנים את `els-code.template.html` (זה המקור — ~190KB, כל ה-HTML/CSS/JS),
  ואז `python3 build.py`. **לא לערוך את `public/tzofen.html` ישירות** (בנוי + דחוס).
- `data/` = נתונים מוטמעים: `tk-letters.txt` (אותיות התנ״ך המנורמלות), `tk-vtext.txt`
  (טקסט-פסוקים), `tk-vlens.txt`/`tk-vchap.txt`/`tk-meta.json` (אינדקס-פסוקים מיושר),
  `niqqud-compact.txt` (ניקוד-תורה), `els-logo-b64.txt` (לוגו ELS של האתר).
- מקורות-הנתונים המקוריים: `public/tanakh-letters.txt` + `public/tanakh-verses.json`
  (התורה יושרה למהדורת-304,805; הנ״ך 1:1). ראה `work_log` לפרטי-היישור.
- **`tk-vtext.txt` (תצוגת-הפסוק) נבנה מחדש ממקור נקי (Sefaria/WLC):** במקור המקף (־)
  נמחק בלי רווח → מילים נדבקו («כלעדת בניישראל»). התיקון: טקסט-פסוקים נקי מיושר
  לגבולות-הפסוק הקיימים (זרם-האותיות אומת זהה 39/39 ספרים → אפס-סחף). משפיע רק על
  תצוגת-הפסוק בחלונית, לא על החיפוש/הדילוגים. הסקריפט: `scratchpad/rebuild_vtext.py`.

## פיצ׳רים (נכון ל-14.7.2026)
- 3 מצבי-תצוגה: מלכותי · **קלף** · קלאסי.
- היקף: 📖 תורה (מהיר) · 🔒 **כל התנ״ך נעול לרשומים** (בלי טעינת-כוח עד שנפתח).
- עיגון-פסוקים (הקשה על אות → ספר·פרק·פסוק + טקסט), חיפוש-מוצלב (לרשומים), עדשות/מתקדם (אדמין),
  PNG · סרט-הצופן (כל פתיחת-צופן מתנגנת כסרט; ביקור-ראשון ×2 על «תורה קדשה»), שיתוף, גלריה.
- **🎓 שער-הדרכה חובה + מאמן-שיטה (21.7.2026):** הנתונים הראו שאנשים מקלידים משפט שלם כמחרוזת-דילוג (ממוצע 10.9 אותיות!) → «לא נמצא» → מתייאשים. התיקון: (1) **שער-הדרכה חד-פעמי** «איך מחפשים נכון» (`openOnboard`, flag `tzofen_onboarded_v1`) נפתח לפני החיפוש הראשון וחוסם אותו — 3 צעדים: מילה קצרה 2־5 אותיות · דילוג נמוך · **הצלבה = «החיפוש המוצלח»**; (2) **מאמן-קלט חי** (`updateCoach`) — משפט/מילה-ארוכה → רמז מתחת לשורה + כפתור «🔀 פצל והצלב» שמפצל לשני שדות (`splitToCross`); (3) ברירת-המחדל «יום משיח בא» הוסרה — הכלי כבר לא מלמד את ההרגל הרע (placeholder+שבבים = מילים קצרות).
- **שער-גישה:** לא-רשום = **קריאת ההדרכה** + `FREE_DEMO=3` חיפושי-מילה לטעימה · **מוצלב + כל התנ״ך + שמירה = לרשומים** («החיפוש המוצלח» דורש רישום) · אדמין = הכל.
- **שמירת-מטריצה = לענן בלבד** (בלי שמירה-למכשיר): «💾 שמור בגלריה» → `postMessage(save)` → `TzofenEmbed` קורא ל-`save_els_matrix` → `els_records`. אדמין → `published` מיד · משתמש רשום → `pending` (אישור בדשבורד) · אנונימי → שער-הרשמה. הגלריה נפתחת מכפתור «🖼️ מטריצות שמורות» (`SavedMatricesGallery`, קורא `published`).
- **אדמין:** 5 הקשות על הכותרת / `#admin` בכתובת → עריכת-צפנים, קשר-לפוסט.
- **דשבורד:** טאב «🔠 הצופן» ב-`AdminPage` (`src/components/ElsStatsTab.jsx`) קורא מ-`visitor_events`.

## תוכנית שלב ב׳ (עתיד)

(טיוטה/מאומת/ציבורי) · תגיות · קישורי פוסט/פסוק/מספר/מסע · גרסאות. הפרטים ב-`work_log`.
